A library to craft modular, composable, and testable command line applications

Craft It

~/www/my-cli-app/cmd.js
var testCommand = command(
  alias('test', 't'),
  handler(function (context) {
    console.log('Testing cli.js!')
  })
)

var testFlag = flag(
  alias('--test', '-h'),
  handler(function (value, context) {
    console.log('Flag value:', value)
  })
)

run(
  process.argv.slice(2),
  app(
    testCommand(),
    testFlag()
  )
)

Execute It

~/www/cli-foo $: _

Why Cli.js? What's different?

There are already several great libraries on NPM that allow developers to create interactive and descriptive command line apps using Node.js (Yargs, Nash, Commander). But, Cli.js's unique modular and composable api promotes small modules that just work, and work together. Everything is a function and everything composes.

See Examples

Modular

Everything is a function and in its own module. Use only what you need, reducing boot up time when executing large or small command line apps.

Composable

Write all your commands and flags as smaller, modular apps. Then compose them together to create your full command line app. Can you say plugins?! Woot!

Testable

Because modularity and pure functions are encouraged with Cli.js, it's easy to create isolated tests in order to boost confidence across your app releases.
